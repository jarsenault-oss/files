Financial impact of a Zillow user data breach

It only takes one compromised employee credential to open the door. What happens next — how many Zillow users' personal data gets exposed — determines the financial liability. Select a user exposure scenario to model the penalty impact across all 19 active state privacy laws.

The threat vector we're solving for
Zillow's audit logs currently have no reliable way to detect when an employee login IP is geographically impossible — a primary indicator that credentials have been stolen. A single compromised account with broad system access can silently exfiltrate personal data on thousands or millions of users before anyone notices. The question isn't if a credential gets stolen — it's how much exposure occurs before it's caught.

Step 2 — State breakdown

Exposure by state

One compromised employee credential can expose millions of users. Penalties are calculated across all active state privacy laws, scaled to each state's share of the US population. California is the only state with a consumer private right of action — all others are AG-enforced per-violation.

How user counts are estimated by state: The number of affected users per state is calculated by multiplying the total scenario user count by each state's share of the US population (US Census 2023). For example, if 1M users are exposed and California represents 11.6% of the US population, the model estimates ~116,000 affected CA users. This assumes Zillow's user base is distributed proportionally to the general US population. In practice, Zillow's users likely skew toward high-activity real estate markets — California, New York, Texas, Florida — meaning actual exposure in those states could be higher than shown here. This model is therefore conservative on state-level distribution.
