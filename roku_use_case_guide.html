Identifying Shared IPs
at CTV Scale

Why the home/business flag isn't enough — and how IPC, Nodify, ASN, and NAT Detector work together to solve it.

Roku's Stated Objective

Roku's product team needs to flag IP addresses very likely shared by multiple households. The home/business flag in NetAcuity has been the go-to, but it has a critical blind spot: a residential ISP using CGNAT returns "home" while that single IP simultaneously serves dozens or hundreds of households. The home flag signals connection context — not IP exclusivity.

Feature Code 9 (home/business) is a binary field describing what kind of entity uses the IP — residential vs. commercial. It doesn't tell you how many entities are behind it.

Current Approach

Home / Business Flag Only

  • Returns "home" for CGNAT IPs serving 100+ households
  • No visibility into device count behind the IP
  • No signal on IP stability or location movement
  • Can't distinguish a single-household from a NAT pool
  • Silent on residential proxies masking multi-user traffic
Recommended Approach

IPC + NAT Detector + Nodify + ASN

  • IPC device count directly surfaces multi-device shared IPs
  • NAT Detector explicitly flags CGNAT-assigned IPs
  • IPC distance range confirms whether devices are co-located
  • Nodify flags residential proxies masking multi-user traffic
  • ASN validates network ownership and routing context
💡
The CGNAT blind spot: Carrier Grade NAT adds a second layer of address translation at the ISP level, hiding entire neighborhoods behind one public IP. CGNAT doesn't change the home/business classification — those IPs still register as residential. IPC's ipc-distinct-device-ids, paired with a tight ipc-dist-std-dev, is the clearest signal that you're looking at a CGNAT pool rather than a single household.

IPC — Key Fields for Shared IP Detection

IPC observes mobile device-derived data (D3) over a rolling 5-week window across 600–700M active IPs monthly. These fields form a behavioral fingerprint for whether an IP is truly single-household or pooled.

ipc-distinct-device-ids
Unique devices observed in 5 weeks. High values (50+) strongly indicate CGNAT or proxy pooling. The primary shared-IP signal.
ipc-dist-std-dev
Std deviation of observed distances. Tight + many devices = co-located CGNAT. Wide + many devices = VPN/proxy routing across regions.
ipc-avg-dist
Average distance between observed locations. Low + many devices = neighborhood CGNAT. High = VPN or proxy routing.
ipc-weeks-stable
Consecutive weeks at the same location. Unstable IPs (0–2 weeks) are dynamically reassigned and more likely shared.
ipc-distinct-postals
Unique postal codes seen. Multiple postals + many devices = strong CGNAT indicator spanning neighborhoods.
ipc-score
Composite score 1–5. Score 5 = high devices + many locations + low stability. Score 1 = dedicated single-household.

NAT Detector — CGNAT Connection Type Flag

Built into NetAcuity Pulse, NAT Detector adds a nat value to the connection type field when an IP is identified as operating under Network Address Translation. The most explicit, direct indicator that an IP is not assigned to a single household.

pulse-conn-type = "nat"
Returned when the IP operates under NAT/CGNAT. Directly flags this IP as shared. Treat geolocation with same caution as mobile IPs.
pulsemax-conn-type
Same NAT indicator in the Pulse Max tier. Connection speed values are unchanged — only the connection type field is affected.
⚠️
NAT IPs aren't limited to one household — they can serve multiple households across nearby towns. Geolocation precision is reduced and should be treated similarly to mobile IPs for targeting decisions.

Nodify — Residential Proxy Detection

Catches a distinct but related problem: IPs classified as residential by ISPs that are actually routing anonymized traffic for multiple users via proxy networks. These blend into legitimate residential ranges and are invisible to the home/business flag.

residential
"yes" = this IP is a residential proxy exit node. Looks like a home connection but serves multiple anonymous users. Very high sharing likelihood.
category
VPN, Proxy, Darknet, Hosting, Consumer-Privacy. A hosting or public proxy category alongside a home classification = near-certain multi-user IP.
last-seen
Timestamp of last confirmed proxy activity. A recent last-seen = proxy is actively in use — unreliable for any single-household attribution.
mobile
"yes" = proxy traffic on a mobile connection, adding another layer of complexity to CTV household attribution.

ASN — Network Routing Context

ASN validates the network-level identity of an IP. For shared IP detection, it confirms whether a "residential" IP is truly on a residential carrier or on hosting infrastructure that shouldn't be receiving residential classifications.

asn
Autonomous System Number. Known residential carrier ASNs (Comcast AS7922, Charter AS20115) vs. hosting providers surface classification mismatches.
asn-name
Human-readable carrier name. Mismatches between asn-name and ISP name can surface sub-allocated or proxy-injected address blocks.
🔍
When an IP returns a residential ISP name but its ASN resolves to a hosting or datacenter provider, that mismatch is a strong signal the IP has been sub-allocated or routes through proxy infrastructure — even if Nodify hasn't flagged active proxy activity yet.
